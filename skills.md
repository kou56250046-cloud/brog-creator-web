# 記事作成スキルガイド

> **重要:** 記事を書く前に必ずこのファイルを最初から最後まで参照すること。

---

## 基本仕様

| 項目 | 内容 |
|------|------|
| 文字数 | **5,000〜10,000字**（本文のみ、frontmatter除く） |
| 言語 | 日本語 |
| 保存先 | `content/articles/YYYY-MM-DD-slug.md` |
| frontmatter | 下記フォーマット参照 |

### frontmatter フォーマット

```yaml
---
title: 記事タイトル（40字以内）
description: 記事の説明（OGP・一覧カード用・120字以内）
date: "YYYY-MM-DD"
category: カテゴリー名
tags: [タグ1, タグ2, タグ3]
status: published
---
```

## バズる記事の本質

記事には2つのアーキタイプがある。書く前にどちらで書くかを決める。

### A型（ハウツー・心理解説系）

**「共感で始まり、感動で終わる」**

読者の感情を先に設計してから逆算して文章を書く。テクニックより先に「この記事を読んだ読者はどんな感情で終わるか」を決める。

**黄金比:** ベネフィット（1位）> 根拠（2位）> 限定性（3位）

### B型（思想・哲学系）

**「衝撃で始まり、問いで終わる」**

読者が信じていた「常識」を解体し、解決策を渡さずに問いを残す。「認識が変わること自体が、行動の変化より先に来る」という信頼のもとに書く。

**構造:** 幻想の提示 > 幻想の解体 > 本当の問いを置く

> 解決策を渡さない記事が、最も長く読者の頭に残ることがある。

## タイトルの作り方

### 必須要素（最低1つ以上）

- **数字**: 「7時間」「3つの」「20人で」「9ヶ月」
- **体験・驚き**: 「〜してみたら」「なんと〜だった」「〜で気づいたこと」
- **読者の悩みをそのまま**: 「眠れない」「やめられない」「なぜ〜」
- **逆説・反常識**: 「〜なのに〜」「〜しても〜ない理由」

### 4つの勝ちパターン

| 構造 | 公式 | 具体例 |
|------|------|--------|
| **A: 状況＋意外性** | [読者の体験] + [予想外の原因] | 「眠れない夜、頭が止まらない理由——意志の力では解決できない」 |
| **B: 数字＋体験＋結果** | [期間/数字] + [行動] + [意外な結果] | 「3ヶ月で月収10万。会社員が朝4時に起きた理由」 |
| **C: 否定＋真実の提示** | 一般論の否定 + [本当の原因] | 「努力が報われないのは、努力の方法が間違っているからじゃない」 |
| **D: 悩みそのまま＋価値示唆** | [読者の状態をそのまま] + [価値の示唆] | 「人間関係でいつも疲れる人へ。HSPと診断される前に知っておくこと」 |
| **E: 逆説・解体型**（B型記事専用） | [一般的な解決策を逆転する] or [逆説的な真実] | 「可能性を狭めると、自由になれる」「やめると、前に進む」「解決しようとするから、解決しない」 |

### タイトル禁止パターン

- 「〜について」「〜の方法」で終わる（検索ワードそのまま）
- 「完全ガイド」「まとめ」系
- カタカナ専門用語から始まる
- **40文字超え**（表示で切れる）
- 「〜の3つの方法」「5ステップで〜」（解決策の番号化を予告するタイトル）— B型記事では特に禁止

## 感情設計（書く前に必ず決める）

### A型（ハウツー・心理解説系）

| タイミング | 読者が感じること |
|------------|----------------|
| 冒頭 | 「あ、これ私のことだ」（共感） |
| 中盤 | 「なるほど、そういうことか」（納得・驚き） |
| 末尾 | 「やってみよう」「誰かに教えたい」（行動・感動） |

### B型（思想・哲学系）

| タイミング | 読者が感じること |
|------------|----------------|
| 冒頭 | 「これは自分だけじゃなく、みんなの問題だったのか」（構造的共感） |
| 中盤 | 「自分が信じていたことが、逆に状況を悪化させていたのか」（幻想の解体） |
| 末尾 | 「答えはわからないけど、この問いを持ち続けたい」（開かれた思索） |

## 記事構成

### A型：解決型フロー（6ステップ）

A型記事（ハウツー・心理解説系）はこの6ステップで構成する。

### Step 1: リード（冒頭200字以内）

**パターンA: 体験描写**
読者が「自分のことを書いている」と感じるシーンを1〜2行で描写。主語を「あなた」にせず情景で語る。

```
例：
「眠いのに、眠れない。
横になるたびに頭が動き出す。
なぜかはわからないけど、これが毎晩続いている。」
```

**パターンB: 驚きの事実**
数字や逆説を使った「えっ」となる一文で始める。

```
例：
「睡眠不足が続くと、食欲を抑えるホルモンが40%低下する。
つまり、寝不足の翌日に食べ過ぎるのは、意志の弱さではなく生理現象だ。」
```

### Step 2: 問題の本質を一言で定義

冒頭の共感・驚きを受けて「この問題の本質はこれだ」と断言する。長くても2〜3文。記事の軸になる。

### Step 3: 原因分析（なぜ起きるのか）

- 読者が「なぜ？」と思っている部分を先回りして答える
- 研究・データ・メカニズムで説明する
- 「一般的に言われていること」vs「実際のメカニズム」を対比させる

### Step 4: 差別化（なぜ一般的な解決策が機能しないのか）

- 表面的なアドバイスが効かない理由を示す
- ここで読者は「この記事は違う」と感じる

### Step 5: 解決策・実践ステップ

- 具体的で今日からできる内容
- 番号リストで整理
- 根拠（論文・研究・データ）を必ず示す

### Step 6: 締め（感動で終わる）

やってはいけないこと：まとめっぽく羅列して終わる  
やること：読者への問いかけ・気づき・背中を押す一言で締める

```
例：
「あなたが「なぜ眠れないのか」と自分を責めてきたなら、
それはもう今日で終わりにしていい。
知ることが、最初の一歩になる。」
```

---

### B型：幻想解体フロー（5ステップ）

B型記事（思想・哲学系）はこの5ステップで構成する。**解決策は提示しない。**

#### Step 1: 衝撃の数字・事実（説明なし）

最初の1〜2文で核心を叩きつける。「なぜなら」「というのも」といった説明句は不要。事実だけを置く。

```
例：
「人の平均寿命は短い。ものすごく、バカみたいに短い。
80歳まで生きるとして、あなたの人生はたった4000週間だ。」

「頑張るほど、仕事が増える。
これは怠惰の問題でも、能力の問題でもない。」
```

#### Step 2: 「〜すれば解決できる」という幻想を提示する

読者が現在信じている解決策を、公平・丁寧に紹介する。批判せず、「確かにそう思うのは自然だ」と認める。

#### Step 3: 幻想がなぜ生まれたかを歴史・構造で説明する

個人の責任ではなく、社会・歴史・経済の力学が原因であることを示す。「あなたのせいではない」が伝わるように書く。

```
例：
「産業革命以前、農民は時計ではなく仕事のリズムで一日を過ごしていた。
「時間を管理する」という発想自体が、近代の発明だ。」
```

#### Step 4: 幻想が問題を悪化させるメカニズムを示す

「解決しようとするほど、問題が深くなる」という逆説の論証。具体的な研究・歴史事例・日常の観察で裏付ける。

#### Step 5: 解決策を渡さず、問いを置いて終わる

「だから〜すれば万事解決する」という締め方はしない。問いを残す。「でも悲しまないでほしい。それが実は、良い知らせだ」という逆転で締めることもできる。

```
例：
「解決策は、ない——少なくとも、あなたが探しているような解決策は。
でもそれは、悪い知らせではないかもしれない。
問いを持つことが、最初の一歩になる。」
```

## 文体・リズム

### 基本トーン

- **専門性は高く、言葉は平易に**
- 「研究者が友人に説明するように」書く
- 断定する（「〜だ」「〜である」）が、押しつけない

### 専門用語の平易化ルール

**中学生テスト**：用語を書いたら「中学生に口頭で説明できるか？」と問う。できなければ即座に日常語に置き換える。

**書き方の順番**：初出時は「平易な言葉（専門用語）」の順で書く。2回目以降は平易語だけでよい。

```
✅ 例：「脳の警報装置（扁桃体）が反応すると〜」
❌ 例：「扁桃体の過活性化により〜」
```

**1記事につき専門用語は最大4つまで**。それ以上は必ず平易語に統一する。

**変換テーブル（代表例）**：

| 専門用語 | 平易な言い換え |
|---------|-------------|
| デフォルト・モード・ネットワーク（DMN） | 脳が「ぼーっとしているとき」に動く回路 |
| 腹側線条体 | 脳の「報酬センター」（快楽を感じる部位） |
| コルチゾール | ストレスホルモン（緊張すると出て体を戦闘態勢にする物質） |
| 扁桃体 | 脳の「警報装置」（危険を感じると鳴り響く） |
| ドーパミン | 「もっと欲しい」を作り出す物質 |
| 前頭前野 | 「冷静に考える」脳の部位 |
| 頭頂葉 | 「自分と外の世界の境界線」を管理する脳の部位 |
| 海馬 | 脳の「記録係」（記憶を保存する部位） |
| 視床 | 感覚情報の「仕分けセンター」 |
| セロトニン | 「穏やかさ」を作り出す物質 |
| ノルアドレナリン | 緊急時に集中力・覚醒を引き上げる物質 |
| 自律神経 | 心臓・呼吸・消化など、意識せずに動く体の制御システム |

### 著者当事者スタイル

著者自身も問題の当事者として語る。説教しない。

```
✅「私もかつて〜だった。」「自分でもこれに気づいたのは最近のことだ。」
❌「あなたは〜しがちです。」「多くの人は〜してしまいます。」
```

著者が問題の外側から語ると読者は「裁かれる側」になる。著者が問題の内側にいることで、読者は「一緒に考える仲間」になる。B型記事では特に重要。

### 段落リズム

```
短い（1〜2行）
　↓
少し長い（3〜4行）
　↓
また短い（1行）
```
この波を繰り返す。長い段落が連続すると読者が離脱する。

### 一行パンチ（核心の一行）

長い論証の後、最も重要な結論を**1行だけの段落**として独立させる。

```
例：
（4行の論証）
（3行の具体例）
（2行の展開）

生産性とは、罠なのだ。         ← 1行だけ独立
```

前後に何も置かない。この「空白に挟まれた1行」が記事全体の軸になる。B型記事で特に効果が高い。

### ダッシュ（——）による転換

読者が「次はこういう話だろう」と思った直後に、反対方向に転じるときに使う。

```
✅「時間管理をうまくやれば、人生は豊かになるはずだ——本当に？」
✅「これが正しい生き方だと思っていた。——でも、そもそも「正しい生き方」とは何なのか」
✅「努力すれば報われる——それは事実か、それとも物語か」
```

### 禁止表現と代替

| 禁止 | 代替 |
|------|------|
| 〜してみましょう | 〜が有効だ / 〜することで〜が変わる |
| 大切です | 〜が核心になる / 〜が鍵だ |
| 心がけましょう | 〜を意識するだけで〜が変わる |
| 〜かもしれません | 〜という研究がある / データは〜を示している |
| ぜひ〜してください | （削除して断定に変える） |
| 〜と思いませんか？ | 事実を述べた後、問いは体言止めで置く |
| 著者が問題の「外」にいる語り口 | 「私も〜だった」で当事者として入る |
| 「〜の3つの方法」「5ステップで〜」（本文中の解決策リスト） | 論証の流れで説得する（B型記事では番号リストは使わない） |

### 見出しの作り方

「答え」ではなく「問い」か「驚き」にする。

```
❌ 睡眠不足とホルモンの関係
✅ なぜ寝不足だと翌日食べ過ぎるのか

❌ 解決策
✅ 意志の力より先に、睡眠を直す

❌ まとめ
✅ 知ってしまったら、もう元には戻れない
```

B型記事（思想・哲学系）では、さらに逆説・皮肉・概念の再定義型を使う：

```
✅ 逆説型：「可能性を狭めると、自由になれる」
✅ 問い返し型：「やり遂げよう。でも、何を？」
✅ 概念の再定義型：「便利さは何を奪うのか」「効率化という罠」
✅ 皮肉な命名型：「人生のベルトコンベア」「忙しさという依存」
```

章内の小見出しも「説明ラベル」にしない。小見出し自体が読者の思考を揺さぶるものにする。

## 学術多角照射（積み上げ収束構造）

複数の学術視点をリレー形式でつなぎ、すべてが同一の結論を指すように配置する。

```
[視点1: ○○学] ○○学的に見ると——（驚きの事実・1〜2段落）
               ↓ つまり〜だ。
[視点2: △△学] さらに、△△学の観点からも同じことが言える——
               ↓ つまり〜だ。
[視点3: □□学] そして□□学もこれを支持する——
               ↓
[収束] ○つの異なる学問が、同じ答えを指している。
```

### 収束構造のルール

| ルール | 内容 |
|--------|------|
| 各視点は200〜400字 | 深く掘り下げない。驚きを一撃で届ける |
| 導入フレーズ | 「○○学的に見ると」「○○の観点では」で始める |
| 橋渡し文 | 各視点の末尾は「つまり〜だ」で結論と接続 |
| 接続フレーズ | 次の視点冒頭は「さらに」「別の角度からも」「同様に」 |
| 収束の宣言 | 「複数の学問が同じ答えを指している」で締める |

### 13分野リファレンス

| 分野 | 驚きの切り口 | 有効なテーマ |
|------|------------|-------------|
| **認知科学** | 脳の情報処理限界・認知バイアス | 意思決定、習慣、先延ばし |
| **行動経済学** | 人が非合理な選択をする構造的理由 | 節約できない、やめられない |
| **神経科学** | 神経伝達物質・脳回路が行動を支配 | 睡眠、依存、ストレス、集中力 |
| **内分泌学** | ホルモン連鎖が感情・行動を物理的に支配 | 睡眠、食欲、疲労、ストレス |
| **統計学** | 信じられていた相関が実は偽り | 健康法、ダイエット、勉強法 |
| **データサイエンス** | 大規模データが明かす反直感的パターン | SNS、購買行動、習慣 |
| **化学** | 日常が実は分子レベルの反応 | 食品、薬、カフェイン |
| **量子化学** ⚠️ | 電子軌道が「見えない差」を生む | 薬の効き方、素材の性質 |
| **量子物理学** ⚠️ | 観測が対象を変えるという逆説 | 自己認識、意思決定 |
| **社会学** | 個人の行動は社会構造の産物 | 人間関係、同調圧力、孤独 |
| **人類学** | 現代の悩みは数万年前から変わらない | 承認欲求、恐怖、コミュニティ |
| **考古学** | 遺跡・遺物が現代行動の起源を示す | 集団行動、儀式、睡眠パターン |
| **量子哲学** ⚠️ | 「確定しない」「観測で変わる」という問い | 自己変革、決断、可能性 |

⚠️ = **比喩・知的驚き専用。事実としての主張には使わない。同じ記事に量子系を2つ以上使わない。**

### 分野選定基準

1. **直接関連度（必須1〜2分野）:** キーワードに最も直結する分野を必ず選ぶ
2. **意外性（1〜2分野）:** 読者が「この分野まで関係あるの？」と感じる分野
3. **収束しやすさ:** 全分野が同一結論に向かえることを確認

## 斜め上の視点（反常識・意外な切り口）

読者が「へえ、そんな見方があったのか」と感じる視点を意図的に設計する。**記事1本につき最低1箇所、必ず入れること。**

### 原則：「当たり前の裏側」を探す

どんなテーマにも「常識とされていること」がある。その裏を探す。

> 「なぜ〜するのか」ではなく、「なぜ〜するように設計されているのか」と問い直す。

### 斜め上視点の5パターン

| パターン | 問い直し方 | 例 |
|---------|------------|---|
| **① 時間軸をずらす** | 「現代の問題 → 実は数万年前から」 | 「孤独感は現代病ではなく、150人の村が消えた進化の傷跡だ」 |
| **② スケールを変える** | 「個人の問題 → 脳・細胞・社会レベルの話だった」 | 「集中できないのはあなたの問題ではなく、スマホが引き起こす神経レベルの構造変化だ」 |
| **③ 因果を逆転させる** | 「〜だから〜になる」を「〜になるから〜だ」に反転 | 「やる気があるから行動するのではない。行動するからやる気が生まれる」 |
| **④ 悪役を主人公にする** | 「ネガティブな感情・現象 → 実は必要な機能だった」 | 「先延ばしは怠惰ではない。未知のリスクを回避しようとする脳の安全装置だ」 |
| **⑤ 別分野の知見で見る** | 「〜の問題を、まったく違う分野で解読する」 | 「人間関係の疲れは、生態学的な『ニッチの重なり』で説明できる」 |

### 具体的な変換例（テーマ「先延ばし」）

```
❌ 普通の視点：
「先延ばしは意志が弱いせいだ。改善するには習慣化が必要」

✅ 斜め上の視点（④悪役を主人公にする）：
「先延ばしは、脳の"進化的な安全策"だ。
やったことのないことに向かう前に「本当に安全か？」と確認する機能が、
現代のタスクにも誤作動している。
先延ばしが激しいほど、あなたの脳の危険察知システムは正常に動いている。
問題はあなたではなく、設計のミスマッチだ。」
```

### 斜め上視点を設計するときの問いかけ

記事のテーマが決まったら、書く前に以下を自問する：

1. この現象は「いつから」始まったのか？（時間軸）
2. 個人のせいにされているが、実はどのレベルの話か？（スケール）
3. 原因と結果を逆にしたら、どう見える？（因果逆転）
4. 「悪いもの」として扱われているが、どんな機能が隠れているか？（悪役の再評価）
5. 全く違う分野の専門家が見たら、どう説明するか？（別分野）

## 実例・事例の使い方

記事に「実際に起きた話」を入れることで、読者は「自分にも起きうる」と感じ、記事への信頼と共感が大きく上がる。

### 実例の種類と使い方

| 種類 | 使い所 | 例 |
|------|--------|-----|
| **実在する人物の事例** | 問題提起・中盤の具体化 | 「哲学者のウィリアム・ジェームズは〜」 |
| **研究で使われた被験者の話** | 原因分析・データの説明 | 「この実験では、参加者の〜%が〜と回答した」 |
| **歴史的事件・出来事** | 学術照射の裏付け | 「1980年代のある調査では〜」 |
| **普遍的な日常シーン** | 冒頭の共感・読者の状況描写 | 「深夜2時、眠れずにスマホを見ている」 |

### 実例を書くときのルール

- **固有名詞＋具体的状況** で書く（「ある人が〜した」ではなく「〜大学の研究者Aは〜した」）
- 実例は**1段落＝1つ**。詰め込まない
- 実例の直後に「だから〜だ」と結論に接続する
- 公人・著名人の実例は事実に基づく。創作・推測を混ぜない

### 実例の配置パターン

```
[冒頭] 日常シーンの描写（読者が「自分のことだ」と思う）
[中盤] 研究事例 or 歴史的事件（学術照射の裏付けとして）
[解決策] 実践した人の変化（「〜を試みた人は〜が変わった」）
```

### たとえ・メタファーの強化ルール

抽象的・専門的な概念を説明するとき、必ず日常にあるものに例える。**1テーマ1たとえ必須**。

**たとえの引き出し（使いやすいドメイン）**：

| ドメイン | 例 |
|---------|---|
| 建物・構造 | 「〜はダムのようなもので」「〜は土台が腐った家のように」 |
| 料理・食事 | 「〜は素材より調理法の問題だ」「スパイスを加えすぎると料理が台無しになるように」 |
| 天気・自然 | 「〜は台風の目のようなもので」「地面の下で水脈が流れるように」 |
| 道路・交通 | 「〜は信号が壊れた交差点みたいな状態だ」 |
| 機械・道具 | 「〜はエンジンのレッドゾーンを超えて走り続けるようなものだ」 |
| 演劇・映画 | 「〜は脚本のない舞台で演じているようなもの」 |
| 動物・生態系 | 「〜は天敵のいない島で増えすぎた草食動物のように」 |
| 子ども・成長 | 「〜は転び方を覚えるまでに何千回も転ぶ子どものようなものだ」 |

**良いたとえの3条件**：
1. 読者が「見たことある / 経験したことある」ものを使う
2. たとえ話の後に「つまり〜だ」で即座に本題に戻る（たとえ話で終わらない）
3. 長くしない（2〜3行以内）

**変換例（Before → After）**：

```
❌ Before（専門的・伝わりにくい）：
「扁桃体の過活性化によって、前頭前野のトップダウン制御が抑制される」

✅ After（たとえで直感的に）：
「脳の警報装置が鳴りっぱなしになると、冷静に考える部分が機能停止してしまう。
火災報知器が鳴り響く中で落ち着いて計算しろと言われても無理なのと同じだ。
パニック中に「落ち着け」が効かないのは、意志の問題ではない。」
```

## 文献・論文の引用

引用があることで「この記事は調べて書いている」という信頼感が生まれる。**1記事につき最低3件の文献を引用する**こと。

### 引用の検索方法

記事執筆前に **WebSearch ツール**を使って実際の論文・文献を検索する。

```
検索クエリの例：
- "prayer meditation neuroscience brain study site:pubmed.ncbi.nlm.nih.gov"
- "synchronicity Carl Jung psychology research"
- "circadian rhythm sleep hormone study"
- "default mode network meditation fMRI"
```

検索後、**WebFetch ツール**で論文のアブストラクトページを取得して著者名・発表年・ジャーナル名を確認する。

### 引用フォーマット

記事本文中での使い方：

```markdown
<!-- インライン引用（本文中） -->
ペンシルベニア大学のアンドリュー・ニューバーグ博士の研究（2001）では、
深い瞑想中に前頭葉の活動が増加し、自己と他者の境界を処理する
頭頂葉の活動が低下することが確認されている。

<!-- 記事末尾の参考文献リスト -->
## 参考文献

1. Newberg, A., et al. (2001). "The measurement of regional cerebral blood flow during the complex cognitive task of meditation." *Psychiatry Research: Neuroimaging*, 106(2), 113-122.
2. Jung, C. G. (1952). *Synchronicity: An Acausal Connecting Principle.* Princeton University Press.
3. Spiegel, D., et al. (2007). "Effects of mindfulness-based stress reduction on sleep quality." *Journal of Psychosomatic Research*, 62, 1-9.
```

### 引用の強度レベル

| レベル | 使い方 |
|--------|--------|
| **強（査読論文）** | 「〜大学の〜博士の研究（発表年）では〜が確認されている」 |
| **中（機関レポート）** | 「〜研究所の報告によると〜」 |
| **弱（書籍・概説）** | 「〜が著書『〜』で述べているように」 |
| **哲学（古典・近代）** | 「ハイデガーは『存在と時間』でこう述べている——」「セネカは『人生の短さについて』でこう語る」 |
| **文学・伝記** | 「カフカは1912年の夏、〜という状況に置かれていた」（固有名詞＋年＋状況で具体性を出す） |
| **歴史的事実** | 「産業革命以前、農民は時計ではなく仕事のリズムで一日を過ごしていた」（出典なしでも事実として通用する歴史的定説） |

B型記事では哲学・文学・歴史引用が学術論文と同等の重みを持つ。「論文がなければ書けない」とは考えない。

### 引用時の注意

- **論文の内容を誇張しない**。「〜が示唆される」程度のものを「証明された」と書かない
- **量子系の論文**は比喩として使う場合でも「これは〜の比喩として理解されたい」と断る
- 見つからない場合は「〜分野の研究によれば（詳細は参考文献リスト参照）」という緩い表現にとどめる
- WebSearch で見つからなかった論文を存在するかのように引用することは絶対にしない

## 解決しない記事の書き方（B型専用）

思想・哲学系テーマでは「解決策を提示しない記事」が読者に最も長く残るインパクトを与える場合がある。

### 5つのルール

1. **読者を責めない** — 問題は個人の意志や努力の不足ではなく、構造・歴史・社会設計の問題として提示する。「あなたのせいではない。設計のせいだ」と示す
2. **解決策を売らない** — 「〜すれば解決する」という約束をしない。「認識が変わることが最初の一歩」と位置づける
3. **問いで締める** — 「あなたはどう思いますか？」ではなく、記事が提示した問いが読者の中に**勝手に残るよう**に設計する
4. **逆説を恐れない** — 「〜を諦めると、〜が手に入る」「〜しないことが、〜の第一歩」という逆説的な結論を臆せず提示する
5. **著者の迷いも見せる** — 「これが答えだ」ではなく「私にもまだわからない。でも、この問いは重要だ」という誠実さが読者との信頼をつくる

### B型に向いているカテゴリー

- 社会・思想（「なぜ〜という構造が生まれたのか」）
- 心理・自己理解（「〜という悩みの正体は、個人の問題ではなく〜だった」）
- 習慣・行動（「〜しようとすることが、〜を遠ざけている」）

## 「見えない現象の論理」カテゴリー専用スタイル

奇跡・シンクロニシティ・予知夢・霊体験・祈り・自然の声などを扱う際のルール：

- **姿勢:** 否定も盲信もしない。「論理的に考えると〜」という中立的な立場を貫く
- **トーン:** 「信じるかどうかより、知ることが大事」
- **構造:** 学術多角照射（積み上げ収束構造）を特に積極的に活用する
- **量子系の活用:** このカテゴリーでは量子物理・量子哲学を比喩として積極的に使ってよい（ただし1記事に1種類まで）
- **禁止:** 「科学的に証明されていない = 嘘」という断定。「信じれば叶う」という根拠なき断定

## 記事作成フロー（毎回この順番で実行）

```
【準備フェーズ】
1. このskills.mdを最初から最後まで参照する
2. カテゴリーを確認・提案する
3. A型 / B型 どちらで書くかを決める
4. 感情設計（冒頭・中盤・末尾の感情）を決める（A型/B型で異なる）
5. タイトル候補を生成する（A型はA〜D構造、B型はE構造を優先）
   - 各2本ずつ計8〜10本生成し、最良を選ぶ
6. 使用する学術分野・引用源（3〜5つ）を選定する
   - B型では哲学・文学・歴史事実も引用源に含める

【リサーチフェーズ】
7. WebSearch で関連論文・文献・哲学書を検索する（最低3件）
   - 検索先: Google Scholar, PubMed, ResearchGate, 書籍, Stanford Encyclopedia of Philosophy
   - 著者名・発表年・ジャーナル名を確認する
   - 必要に応じ WebFetch でアブストラクトを取得する
8. 使用する実例・事例を選定する（最低1件）

【執筆フェーズ】
9. 斜め上の視点を1箇所設計する（5パターンからどれを使うか決める）
10. 見出し構成を組み立てる
    - B型は逆説・問い返し・概念の再定義型の見出しを使う
11. 本文を執筆する（5,000〜10,000字）
    - A型：冒頭に共感描写 → 中盤に学術照射 → 末尾に解決策＋感動
    - B型：冒頭に衝撃の数字/事実（説明なし）→ 幻想の解体 → 問いで終わる
    - 末尾に「参考文献」セクションを追加する
12. frontmatterを付けてMDファイルとして保存する
```
